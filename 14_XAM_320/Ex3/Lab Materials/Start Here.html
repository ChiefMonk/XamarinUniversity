Before you begin

This is a Xamarin.Forms class which includes projects for iOS, Android, and Windows. When working on the exercises, you can select the easiest supported platform to run your application on. Refer to the Setup Requirements on the Xamarin University website if you need help with your development environment.

Overview

This session introduces the Model-View-ViewModel (MVVM) architectural design pattern and shows you how to apply it in your Xamarin.Forms applications using the built-in data binding infrastructure.

Objectives

  1. Decide where to place code in Model-View-ViewModel
  2. Define Visual Behavior
  3. Use Commands
  4. Test MVVM based apps

Lecture

XAM320 - Model-View-ViewModel in Xamarin.Forms.

Exercises

  1. Defining a ViewModel
    Exercise
    In this exercise, we will create a ViewModel and use data binding to connect it to a Xamarin.Forms GUI.
  2. Creating ViewModel Collections
    Exercise
    We will continue our prior exercise and create a MainViewModel to manage a collection of quotes.
  3. Driving behavior through properties
    Exercise
    In this exercise, we will add in selection support, and use data triggers to change the UI based on property values exposed in the view models.
  4. Using commands to run behavior
    Exercise
    The last piece we can control in our ViewModels is the behavior of the application - what happens when you tap the button or swipe that list. In this exercise, we will examine commanding - the official way to push behavior into our bindable ViewModel.
  5. Adding unit tests for View Models
    Exercise
    This demonstration will show an example of providing some unit tests for our MainViewModel along with some dependency injection to isolate the services which are used in the view model.
